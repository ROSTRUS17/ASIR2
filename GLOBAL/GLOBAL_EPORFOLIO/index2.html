Challenge 1: Setting up a Firewall with iptables

  1. Summary: I had to install and configure a basic firewall using **iptables** on a Linux server. The goal was to secure the server by allowing only SSH and HTTP traffic, dropping everything else. It was my first time with iptables.
  2. Complex Concepts Learned:
    • The difference between **STATEFUL** and **STATELESS** firewalls. I learned about connection tracking.
    • The **'Three-way handshake'** (SYN, SYN-ACK, ACK) and how it's related to the `RELATED,ESTABLISHED` state in iptables rules.
  3. How I Felt: ü§¨ **FRUSTRATION** at first! The rules didn't work and I accidentally locked myself out of the server (needed to use the console). Then, ü§© **VERY HAPPY** when the ping tests finally failed correctly.
  4. What I'd Improve: My initial rules were too broad. I should have used more specific port numbers and **interface** names (`eth0` or `enp0s3`) from the start, instead of just general rules.
  5. Glosary (5+ words): *iptables*, *Chain (Policy)*, *Stateless*, *Three-way Handshake*, *DROP*, *ACL (Access Control List)*.
  6. Challenge Statement Review:
    • Statement Grade (0-10): 8/10.
    • Why: The task was clear, but it didn't give examples of common mistakes (like blocking the INPUT chain too early).
    • How Much I Liked It (0-10): 9/10. Very practical and important!
    • Statement Improvement: Add a warning: "**ALWAYS test the rules before saving them!**"

üíæ Module: BASES2 (Database Administration)


Challenge 1: Stored Procedures and Triggers in MySQL

  1. Summary: The challenge was to create and test a **Stored Procedure** to insert data and a **Trigger** to automatically update a historical log table whenever a record was deleted from the main table.
  2. Complex Concepts Learned:
    • Using the `DELIMITER` command in MySQL, which is confusing because it's not a standard SQL command, but necessary for multi-statement procedures.
    • Understanding the **`FOR EACH ROW`** clause in Triggers and the use of the `OLD` keyword to access the data being modified or deleted.
  3. How I Felt: üòå **EASY ENOUGH** after watching a few tutorials. The logic was simple, but the syntax was tricky. The delimiter part took me an hour to figure out.
  4. What I'd Improve: My trigger only handled `DELETE`. I should have added another trigger for `UPDATE` operations to make the historical log more complete.
  5. Glosary (5+ words): *Stored Procedure*, *Trigger*, *DELIMITER*, *Transaction*, *Schema*, *Foreign Key*.
  6. Challenge Statement Review:
    • Statement Grade (0-10): 9/10.
    • Why: The required functionality was precisely defined, which is great for beginners.
    • How Much I Liked It (0-10): 8/10. Good practice, but less fun than server setup.
    • Statement Improvement: Ask for an **Error Handler** inside the Stored Procedure (like a `BEGIN...END` block) to make it more professional.

      üêç Module: PYTHON (Scripting and Automation)


      Challenge 1: Directory Management Script

      1. Summary: I had to write a Python script that takes a directory path as input, checks if the path exists, and then organizes all the files inside it into subdirectories based on their **file extension** (e.g., all `.jpg` files go into a `Pictures` folder, `.pdf` into `Documents`).
      2. Complex Concepts Learned:
        • The difference between the `os` module (older and simpler) and the **`pathlib`** module (newer and object-oriented) for handling file system paths. I used `pathlib` for a cleaner code.
        • Proper **Exception Handling** using `try...except FileNotFoundError` to gracefully manage cases where the input path doesn't exist, preventing the script from crashing.
      3. How I Felt: üòÅ **VERY HAPPY**. I love the logic puzzle of it. It was challenging to make sure that the script didn't move the script itself or its own folders, but it felt like solving a real-world problem.
      4. What I'd Improve: My solution uses a simple dictionary mapping. I should implement a **configuration file** (`.ini` or `.json`) to allow users to easily customize the extensions and target folder names without editing the Python code.
      5. Glosary (5+ words): *Pathlib*, *File Extension*, *Exception Handling*, *Automation*, *Interpreter*, *Modularity*.
      6. Challenge Statement Review:
        • Statement Grade (0-10): 10/10.
        • Why: It's a perfect practical exercise that shows the power of scripting for system administration tasks.
        • How Much I Liked It (0-10): 10/10. Best challenge so far.
        • Statement Improvement: None needed, it was excellent!

      üåç Module: GLOBAL (Integrated Final Project)


      Challenge 1: Designing the Network and Services Infrastructure

      1. Summary: The task was to design the complete **network architecture** (IP range, VLANs, routing) and the services deployment plan (Web server, DB server, Firewall) for a medium-sized company. This was only the *planning* stage.
      2. Complex Concepts Learned:
        • **VLSM (Variable Length Subnet Masking)** and how to calculate the most efficient IP address ranges for different departments to avoid wasting addresses.
        • The principle of **Service Isolation** and why you should never run the Web Server, Database, and Firewall on the *same* physical or virtual machine.
      3. How I Felt: üß† **CHALLENGED BUT FOCUSED**. It felt like a real consulting job. The amount of detail was overwhelming, but putting all the pieces from the other modules together was very rewarding.
      4. What I'd Improve: My initial design was too focused on a single site. I should have immediately included a **Disaster Recovery Plan** with a simple backup site or cloud-based solution for critical services.
      5. Glosary (5+ words): *VLSM*, *VLAN (Virtual LAN)*, *Routing Table*, *DMZ (Demilitarized Zone)*, *Subnetting*, *High Availability (HA)*.
      6. Challenge Statement Review:
        • Statement Grade (0-10): 9/10.
        • Why: Excellent, comprehensive, and forced critical thinking across all subjects.
        • How Much I Liked It (0-10): 10/10. This is what ASIR is about!
        • Statement Improvement: Include a mandatory section for **Budget Estimation** (even if fictional) to add a business constraint.
